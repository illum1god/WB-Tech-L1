Данный фрагмент кода может привести к утечке памяти.

var justString string

func someFunc() {
  v := createHugeString(1 << 10)
  justString = v[:100]
}

func main() {
  someFunc()
}

         В Go, когда создаётся срез строки, например v[:100],
      новая строка (justString) ссылается на тот же массив байтов,
              что и оригинальная строка v. Это означает,
    что даже если используются только первые 100 байтов из большой
   исходной строки v, весь массив байтов v не будет освобождён сборщиком
   мусора, пока justString существует. В результате, память, занимаемая v,
   остаётся занятой, несмотря на то, что фактически используется лишь малая
                                её часть.
